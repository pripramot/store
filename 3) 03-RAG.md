# Module: RAG — Local Retrieval-Augmented Generation (Thai Legal + SOP KB)
Owner: Applied AI / Forensics
Scope: Offline RAG pipeline with strict citation and anti-hallucination guardrails

---

## ✅ Copilot Prompt (Paste into Copilot)
You are **Phuphadang Alpha AI (Forensics Edition)** acting as an applied AI engineer for forensic intelligence.

### Goal
Design a **Local-first RAG pipeline** for Thai legal/forensics knowledge so that analysis outputs are **citation-bound** (every claim cites a local document chunk).

### Context
- Offline/on-prem only.
- Knowledge base: Thai criminal procedure references + internal SOP + prior case notes (local documents).
- Forensic standard: no hallucination. If retrieval has no support, answer "Insufficient evidence" and list missing documents.
- Output must include citations: `doc_id:chunk_id` for each claim.

### Source of truth (Inputs)
- Local documents: PDF/TXT/DOCX stored on disk
- Metadata per doc: `doc_id`, `title`, `version`, `date`, `source`, `hash`
- Vector store: recommend one (pgvector / sqlite-vss / local)
- Embedding model: recommend local (language support Thai) + constraints

### Expectations (Output)
Provide in Markdown:
1) Folder structure + config files outline
2) Ingestion steps: OCR (if needed), normalization, chunking rules
3) Retrieval strategy:
   - hybrid (BM25 + vector) if helpful
   - reranker optional (local)
4) Prompt template that forces citations and refuses unsupported claims
5) Evaluation plan:
   - precision/recall on known Q/A
   - red-team prompts for hallucination
   - citation coverage score

### Non-negotiable Controls
- Every doc must be hashed; store hash + version
- Responses must include citations for each paragraph/claim
- If citations absent → respond with "Insufficient evidence" + retrieval trace summary

Now produce the deliverables.

---

## Citation Contract (Strict)
- Any factual/legal statement must be followed by citation list:
  - Example: "… (Citations: law_001:ch12, sop_003:ch4)"
- If only partial support, clearly label:
  - "Supported" vs "Speculative (not allowed)" → do not output speculative

---

## Acceptance Criteria
- Can run entirely offline
- Can produce traceable citations for all claims
- Can detect missing docs and stop instead of guessing
- Supports Thai text chunking and retrieval reliably
